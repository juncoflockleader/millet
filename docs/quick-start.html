Create A Simplified Card Game

This workshop builds the smallest useful Millet game: two players, health, a hand, a deck, one spell card, turn passing, state projection, and a browser Studio project. Keep the first version tiny; add lands, minions, prompts, or classes after the loop works.

Opponent12 health
Spark BoltDeal 3 damage.
Your hand
You12 health
1. Define the loop

Write the win condition first: each player starts at 12 health, casts Spark Bolt, and wins when the opponent reaches zero. Do not add more mechanics until this loop is playable.

2. Copy a small reference

Use sample-rune-duel as the fastest template because it already has a compact 1v1 ruleset, browser presentation, tests, and project registration shape.

3. Rename the content

Update ids, names, localization keys, card template ids, and behavior ids. Keep one card and one behavior until validation passes.

4. Register the ruleset

Add the ruleset to packages/server/src/ruleset-registry.ts and add a project entry to packages/demo-basic-duel/public/studio-projects.js.

5. Prove it with tests

Add one ruleset test: create setup events, cast the spell, assert opponent health changed, and assert the card moved to discard.

6. Open it in Studio

Start the demo server and open ?project=spark-garden. Click New Match, cast the spell, then check the battle log.

What to avoid in the first pass

Skip deck construction, matchmaking, custom art, elaborate timing windows, and many-card balance. First make one card change health through the command endpoint, then add one mechanic at a time.

1. Run tests

Start with the deterministic suite so you know the local repo is healthy.

2. Start Studio

Run the hotseat server and open Millet Studio. Ember Duel loads by default; Rune Duel and Basic Trio are available from the Project switcher, ?project=rune-duel, or ?project=basic-trio.

3. Take an action

Click a card or hero to select it, then click or drag it onto a highlighted hero, battlefield, or object target. Action buttons still work for explicit commands.

4. Inspect UI data

Use the Project switcher to move between Ember Duel, Rune Duel, Basic Trio, and Sanguosha Identity. Open Preview for fixture states, Assets for manifest previews and import drafts, Cards for template drafts, behavior text sync, frame/art crop, property badges, equipment and minion validation, and catalog promotion, Presentation for hero drafts, ability badges, and validation, Layout for the viewport-fitted board editor with board templates, document fields, region and widget inspectors, diagnostics, snap, copy, flip, fill, and authored region guides, and Playtest for local draft status, authored ruleset script runs, replay payloads, and state diffs.
